# The Agentic Engineering Manifesto

Principles for building systems where humans steer intent, agents execute within governed boundaries, and verified outcomes are the only measure that matters.

---

## Why This Manifesto Exists

The Agile Manifesto was written in 2001 for a world where humans wrote all the code. That world no longer exists. Autonomous agents now generate applications in hours, execute migrations during a single flight, and run verification pipelines that dwarf what any human QA team could attempt in a quarter. The practices built on Agile's assumptions — two-week sprints, story points, daily standups, manual code review — have become structural liabilities, not just dated rituals.

The industry recognizes this. Multiple manifestos, frameworks, and standards have emerged to fill the vacuum — from Casey West's Agentic Manifesto to the P3 Group's *From Sprints to Swarms*, from AWS Prescriptive Guidance to ISO/IEC 5338. Each gets something right. None of them are complete.

**[Beyond Agile: Why the Software Industry Urgently Needs a New Manifesto](beyond_agile.md)** presents the full case: ten structural failures in Agile's assumptions, a critical analysis of every competing framework, and the argument for why the successor must be simultaneously philosophical, principled, operational, and evolutionary.

This manifesto is that successor.

## Core Values

| We value more | over | We also value |
|---|---|---|
| Iterative steering and alignment | | Rigid upfront specifications |
| Verified outcomes with auditable evidence | | Fluent assertions of success |
| Right-sized agent collaboration | | Monolithic god-agents |
| Curated, high-signal context and memory | | Stateless sessions and noisy memory |
| Tooling, telemetry, and observability | | Chat-based heroics |
| Resilience under stress | | Performance in ideal conditions |

That is, while there is value in the items on the right, we value the items on the left more.

## Twelve Principles

1. **Outcomes are the unit of work** — not tasks, not story points, not lines of code
2. **Specifications are living artifacts that evolve through steering** — not frozen documents or absent documentation
3. **Architecture is defense-in-depth, not a document** — layered constraints, not a single gate
4. **Right-size the swarm to the task** — one agent or twenty, chosen by evidence
5. **Autonomy is a tiered budget, not a switch** — earned through demonstrated reliability
6. **Knowledge and memory are distinct infrastructure** — curated differently, governed differently
7. **Context is engineered like code** — budgeted, tested, version-controlled
8. **Evaluations are the contract; proofs are a scale strategy** — deterministic, statistical, formal, human
9. **Observability and interoperability cover reasoning, not just uptime** — trace the why, not just the what
10. **Assume emergence; engineer containment** — probabilistic systems surprise; plan for it
11. **Optimize the economics of intelligence** — the right model at the right cost for the right task
12. **Accountability requires visibility** — every autonomous decision has a human-traceable chain

## The Agentic Loop

**Specify &rarr; Plan &rarr; Execute &rarr; Verify &rarr; Observe &rarr; Learn &rarr; Govern &rarr; Repeat**

Every principle maps to at least one phase. Every phase enforces at least one principle. The loop never terminates — it tightens.
